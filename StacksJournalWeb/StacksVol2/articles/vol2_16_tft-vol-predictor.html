TFT Realized Volatility Predictor: 0.702 Correlation, One Data Leakage Lesson, and a Hybrid Gate Path Forward

StackStrader’s first deep learning model — a Temporal Fusion Transformer trained on 20 years of daily data across 7 tickers — predicts per-ticker forward realized volatility with genuine skill on high-vol names (TSLA, NVDA) but loses to VIX on index-like tickers. The honest result, after fixing a data leakage bug that inflated correlation from 0.988 to 0.702.

Chris · March 27, 2026 · 11 min read

Can deep learning predict per-ticker realized volatility better than VIX? If so, we can identify when to sell covered calls — when implied volatility is overpriced relative to what a given stock will actually realize. The answer, after building and honestly evaluating a Temporal Fusion Transformer (TFT) with 20 input features and a 60-day lookback window, is: sometimes. The model achieves 0.702 Pearson correlation with forward 20-day realized volatility on the test set (2024–2026), versus 0.127 for VIX. But the aggregate number hides a story. TSLA and NVDA show strong TFT advantage (+0.36 and +0.33 correlation lift over VIX). AAPL, QQQ, and MSFT favor VIX. The practical path forward is a hybrid gate: TFT for high-vol individual stocks, VIX for index-like names.

The Question

Covered call profitability depends on the gap between implied volatility (IV) and realized volatility (RV). When IV exceeds future RV, the premium seller collects more than the option is actuarially worth. When RV exceeds IV, the position bleeds alpha through assignment costs and unrealized gains capped at the strike.

VIX is the standard IV proxy, but it is a market-wide measure. It cannot distinguish between TSLA’s idiosyncratic earnings-driven vol regime and AAPL’s low-vol mean-reversion. A per-ticker realized volatility predictor, if accurate, would enable a more precise gate: sell CCs on tickers where the model predicts low forward RV relative to current IV, veto tickers where the model predicts high forward RV.

The Temporal Fusion Transformer (TFT) is the architecture of choice because it handles mixed-frequency inputs (daily OHLCV, weekly macro, static ticker identity), produces quantile forecasts (P10/P50/P90), and exposes variable importance weights that reveal which features drive predictions. This is StackStrader’s first deep learning model, built in ml_studio/ across three files and 880 lines of production code.

The Architecture

The TFT ingests 20 time-varying features across a 60-trading-day lookback window and predicts a single step ahead: today’s forward 20-day annualized realized volatility.

Three known-future covariates — day_of_week, month, and is_opex_week — give the model calendar awareness. OpEx week is computed as the Monday-through-Friday window containing the third Friday of each month, when options expiration concentrates gamma exposure and amplifies intraday volatility.

Static categoricals: ticker identity, enabling the model to learn per-ticker volatility regimes. Seven training tickers: INTC, NVDA, AAPL, AMD, MSFT, QQQ, TSLA. IBIT excluded from training (only 538 rows, listed January 2024) but available for inference.

Training runs on Apple Silicon MPS acceleration via PyTorch Lightning. The temporal train/validation/test split is strict: train on data before 2022-01-01, validate on 2022–2023, test on 2024–2026. No shuffling, no random splits — the test set is entirely out-of-sample and forward in time.

Target: Forward Realized Volatility

The target is fwd_rv_20d: the annualized standard deviation of log returns over the next 20 trading days, computed as std(log_returns[t+1:t+20]) * sqrt(252). This is the quantity a covered call seller needs to forecast — how much the stock will actually move over the option’s remaining life. Three horizons are computed (5d, 10d, 20d) but the primary model targets 20-day RV, matching the typical covered call DTE of 20–30 days.

The Data Leakage Lesson

The first training run produced a 0.988 Pearson correlation between predicted and actual forward realized volatility on the test set. A result that good, on a noisy financial time series, is not a breakthrough. It is a bug.

A 0.988 correlation on financial data is not a model. It is a data leak. Always verify your pipeline before celebrating.

The root cause was a single line in ml_studio/tft_data.py, function make_timeseries_dataset():

time_varying_unknown_reals=FEATURE_COLS + TARGET_COLS

TARGET_COLS is ["fwd_rv_5d", "fwd_rv_10d", "fwd_rv_20d"] — the forward-looking realized volatility targets themselves. By including them as time-varying unknown reals, the TFT received the actual future RV values in the encoder’s lookback window. The model did not need to predict anything. It only needed to copy the most recent target value forward by one step.

The fix (commit 0997e9e) was one line:

time_varying_unknown_reals=FEATURE_COLS

After removing the leak, correlation dropped from 0.988 to 0.673 on the immediate retrain, stabilizing at 0.702 on the final v8 run after training converged. The lesson is not novel — target leakage is the most common ML failure mode in finance — but it is worth documenting because the bug was syntactically invisible. The + TARGET_COLS concatenation looked like deliberate feature engineering. Only the implausible accuracy revealed the error.

The commit also fixed predict_to_parquet() in ml_studio/tft_vol_predictor.py, which had been calling model.predict() in mode="quantiles" and extracting a tensor of shape [N, 3] for P10/P50/P90. After the leakage fix, the quantile output structure changed. The corrected code uses default prediction mode with .squeeze() and estimates P10/P90 from the point prediction via scaling factors (P10 = P50 * 0.7, P90 = P50 * 1.4). This is an approximation; proper quantile extraction requires further work.

On aggregate, the TFT dramatically outperforms VIX as a predictor of forward realized volatility. The 0.702 correlation is strong for a financial forecasting task — for context, typical equity return prediction models achieve 0.01–0.05 correlation. Volatility is more predictable than returns because it clusters (GARCH effects) and mean-reverts, but 0.702 across 7 tickers and 2+ years of out-of-sample data is a genuine result.

The RMSE comparison (0.168 vs 0.467) tells a complementary story. VIX systematically overestimates realized volatility during calm periods — the well-documented variance risk premium (VRP). The TFT learns the per-ticker volatility distribution directly and avoids this structural bias.

The per-ticker breakdown reveals a clean split. The TFT excels on high-vol individual stocks with strong idiosyncratic volatility regimes: TSLA (+0.36 lift), NVDA (+0.33), AMD (+0.22), INTC (+0.16). These tickers have earnings-driven vol spikes, product cycle dynamics, and sector-specific risk factors that per-ticker features (RSI, CMF, ATR, Hurst exponent) can capture but VIX cannot.

VIX wins on index-like and low-idiosyncratic-vol names: QQQ (-0.11), MSFT (-0.04), AAPL (-0.03). This makes structural sense. QQQ is an index ETF — its realized volatility is, by construction, close to market-wide vol. AAPL and MSFT are mega-caps whose vol dynamics are dominated by macro factors (rates, risk appetite) rather than idiosyncratic events. VIX is already a good proxy for their realized vol, and the TFT’s per-ticker features add noise rather than signal.

Quantile Calibration

The TFT was trained with QuantileLoss targeting the 10th, 50th, and 90th percentiles. Calibration measures whether the model’s uncertainty estimates are honest — does the actual value fall below the P10 prediction roughly 10% of the time?

Note: P10/P90 currently estimated via scaling (0.7x, 1.4x P50). True quantile extraction pending.

The P50 at 55% actual coverage is close to ideal. The P10 and P90 are somewhat miscalibrated — the lower tail is too wide (16% vs 10%) and the upper tail is too narrow (83% vs 90%). This means the model slightly underestimates the probability of very high realized vol events. For covered call gating, this is the dangerous direction: underestimating tail vol means the gate may allow selling premium into regimes that are riskier than the model thinks. A caveat: the current quantile estimates use the P50 * 0.7/1.4 approximation rather than native quantile outputs, so calibration numbers will change once proper quantile extraction is restored.

The TFT-based CC gate fires when predicted forward RV is below VIX (IV overpriced, favorable for premium sellers). When the model says “sell,” the win rate is 73.9% — materially higher than the 61.2% win rate on days the model vetoes. The 68.5% signal accuracy means the model correctly identifies the VRP direction (whether realized vol will be above or below implied vol) more than two-thirds of the time.

For comparison, the VIX-based baseline strategy (sell when VIX/100 > historical RV) has no predictive directionality — it uses backward-looking RV, not forward-looking. The TFT’s advantage is that it forecasts where RV is going, not where it has been.

What It Means

The TFT has genuine predictive skill on high-vol individual stocks where per-ticker features — technical indicators, microstructure signals, the Hurst exponent — carry information beyond what VIX encodes. It does not have skill on index-like tickers where market-wide volatility is the dominant factor. This is not a model failure. It is the model correctly identifying which tickers have learnable idiosyncratic vol dynamics.

The practical path for the StackStrader covered call system is a hybrid gate:

  • TFT gate for TSLA, NVDA, AMD, INTC — tickers where the model’s per-ticker features add 0.16–0.36 correlation lift over VIX.
  • VIX gate for AAPL, MSFT, QQQ — tickers where VIX is already a strong proxy and the TFT’s additional features are noise.
  • Default to VIX for any ticker without sufficient training data (e.g., IBIT with only 538 rows).

This mirrors the per-ticker routing architecture already in place for Chronos entry gates (documented in the V15 Chronos Breakthrough article). The TFT gate would operate as a complementary signal — Chronos captures directional momentum risk, while TFT captures volatility regime risk. The two address different failure modes of the covered call strategy.

Technical Details

Implementation

Three files in ml_studio/ comprise the full pipeline:

  • ml_studio/tft_data.py (425 lines) — Feature engineering, forward RV target computation, temporal splits, TimeSeriesDataSet construction. Computes the Hurst exponent via rolling rescaled range (R/S method, 100-bar window), Chaikin Money Flow (20-period), Bollinger Band width, and VIX term structure ratio. CBOE VIX3M and SKEW data are joined from data/macro/cboe_vix_term.parquet when available, with sensible imputation (VIX3M = VIX * 1.05, SKEW = 1.20) for pre-2018 dates.
  • ml_studio/tft_vol_predictor.py (351 lines) — Model definition, training loop, checkpoint management, prediction export. Uses PyTorch Forecasting’s TemporalFusionTransformer.from_dataset() with QuantileLoss, Adam optimizer, and reduce-on-plateau LR scheduling. Training logs (hyperparameters, best val loss, epoch count) are saved to training_log.json alongside each checkpoint.
  • ml_studio/tft_evaluate.py (454 lines) — Evaluation framework with Pearson correlation, RMSE, Black-Scholes CC simulation, and three strategy comparisons (always-sell, VIX-based, TFT-based). The CC simulation prices options using engine.bs_call() with strike = close + ATR_14, T = 20/252, r = 4%, sigma = VIX.

The gate trigger itself lives at scripts/triggers/tft_trigger.py (149 lines), implementing TFTRvSafeAlpha — a stateless trigger that loads pre-computed predictions from parquet, compares the P50 forward RV to current VIX, and returns should_open() = True (safe to sell) when predicted RV < IV. Default is safe (allow selling) when predictions are unavailable.

Data Scale

Training data spans approximately 2006–2022 (16 years), with validation on 2022–2023 and test on 2024–2026. For each of the 7 training tickers, this yields roughly 4,000 daily rows per ticker, or 28,000 total training samples. With a 60-day lookback, the effective training set contains roughly 25,000 input-output pairs. The model converges in 15–25 epochs before early stopping fires, with typical training time of 8–12 minutes on Apple Silicon M-series MPS.

Limitations

Several limitations constrain the current results and should be addressed before production deployment:

  1. Quantile extraction is approximate. The P10/P90 estimates use fixed scaling factors (0.7x, 1.4x) rather than native QuantileLoss outputs. The commit 0997e9e note documents this as a known regression from the leakage fix. Calibration numbers will shift once proper quantile extraction is restored.
  2. Seven training tickers is narrow. The model has seen INTC, NVDA, AAPL, AMD, MSFT, QQQ, and TSLA. Generalization to the remaining 39 tickers in the portfolio is unvalidated. IBIT, with only 538 rows (listed January 2024), cannot be trained at all with a 60-day lookback.
  3. No feature importance reported yet. The TFT’s interpretability mechanism (extract_interpretability() in tft_evaluate.py) is implemented but has not been run on the v8 model. Variable importance weights would reveal whether the model relies on sensible features (rv_20d, atr_14, vix_term_ratio) or latches onto spurious correlations.
  4. CBOE term structure data is partially imputed. VIX3M and SKEW are available from approximately 2018 onward. For 2006–2018 training data, these features use fixed imputation (VIX3M = VIX * 1.05, SKEW = 1.20), meaning the model trains on synthetic macro data for half its history.
  5. The CC simulation uses simplified pricing. Strike is close + ATR_14, a rough OTM proxy. Real covered call strikes depend on delta targeting, available strike granularity, and bid-ask spread. The simulation does not model transaction costs, slippage, or early exercise risk.

The TFT realized volatility predictor is not a silver bullet. It does not replace VIX, and on three of seven tickers it should not try. But on the four high-vol names that matter most for covered call premium — TSLA, NVDA, AMD, INTC — it provides a per-ticker volatility forecast that VIX cannot. The immediate next step is to wire the hybrid gate (TFT for individual stocks, VIX for index-like names) into the V15 experiment loop and measure the marginal alpha over Chronos entry gates alone. The deeper lesson is that the first deep learning model in the system was not held back by architecture or compute. It was held back by a one-line data leakage bug that made the results look too good to question.
